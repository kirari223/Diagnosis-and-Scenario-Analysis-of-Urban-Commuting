## 核心机制

**跳步执行的本质**：不是"继承上一步的内存变量"，而是"从磁盘重新加载上一步保存的文件"。

Jupyter kernel 重启后内存清空，所以中间变量必须落盘。你已有 `save_matrix`/`load_matrix` 和 `get_result_path`，直接用这两个函数实现检查点即可。

---

## 实现模式

### 每个章节函数的标准结构

```python
# main_pipeline.ipynb 中每个章节的 cell 模式

# ---- 章节开关 ----
FORCE_RECOMPUTE = False  # 改为 True 强制重算

# ---- 检查点路径 ----
checkpoint_od = get_result_path("2.Pattern_Analysis/intermediate", "od_matrix.csv")
checkpoint_zones = get_result_path("2.Pattern_Analysis/intermediate", "zone_stats.csv")

# ---- 加载或计算 ----
if not FORCE_RECOMPUTE and checkpoint_od.exists():
    # 跳步：从磁盘加载
    od_matrix = pd.read_csv(checkpoint_od, index_col=0, encoding="utf-8-sig")
    zone_stats = pd.read_csv(checkpoint_zones, index_col=0, encoding="utf-8-sig")
    print(f"已加载中间结果: {checkpoint_od}")
else:
    # 正常计算
    od_matrix = compute_od_from_raw(...)
    zone_stats = compute_zone_stats(od_matrix, zone_gdf)
    
    # 保存检查点
    od_matrix.to_csv(checkpoint_od, encoding="utf-8-sig")
    zone_stats.to_csv(checkpoint_zones, encoding="utf-8-sig")
    print(f"已保存中间结果: {checkpoint_od}")
```

### 跳步执行时的实际流程

```
第一次运行（从头）:
  Cell: 章节2 → 计算 → 保存到 results/2.Pattern_Analysis/intermediate/od_matrix.csv
  Cell: 章节3 → 计算 → 保存到 results/3.Situation_Diagnosis/intermediate/...

第二次运行（只改章节3）:
  Cell: 章节2 → 检测到文件存在 → 直接 pd.read_csv → od_matrix 变量恢复
  Cell: 章节3 → FORCE_RECOMPUTE=True → 重新计算
```

## 不同数据类型的保存方式

你的项目中主要有三类中间数据，对应不同的保存方式：

```python
from src.utils import save_matrix, load_matrix, get_result_path
import geopandas as gpd

# 1. DataFrame（统计表、OD矩阵）
od_matrix.to_csv(get_result_path("2.Pattern_Analysis/intermediate", "od_matrix.csv"),
                 encoding="utf-8-sig")
od_matrix = pd.read_csv(get_result_path("2.Pattern_Analysis/intermediate", "od_matrix.csv"),
                        index_col=0, encoding="utf-8-sig")

# 2. numpy 矩阵（大型数值矩阵，如距离矩阵）
save_matrix(dist_matrix, get_result_path("2.Pattern_Analysis/intermediate", "dist_matrix.npy"))
dist_matrix = load_matrix(get_result_path("2.Pattern_Analysis/intermediate", "dist_matrix.npy"))

# 3. GeoDataFrame（空间数据）
zone_gdf.to_file(get_result_path("2.Pattern_Analysis/intermediate", "zone_gdf.gpkg"),
                 driver="GPKG")
zone_gdf = gpd.read_file(get_result_path("2.Pattern_Analysis/intermediate", "zone_gdf.gpkg"))
```

## main_pipeline.ipynb 的实际结构建议

```python
# ===== Cell 0: 全局开关 =====
RUN_SECTIONS = {
    "1.Data_Preparation":    False,
    "2.Pattern_Analysis":    False,
    "3.Situation_Diagnosis": True,
    "4.Scenario_Computation": False,
}
# 各章节内部是否强制重算（False = 有中间文件就跳过）
FORCE_RECOMPUTE = {
    "1": False,
    "2": False,
    "3": True,   # 当前调试章节
    "4": False,
}

# ===== Cell 1: 章节2 =====
if RUN_SECTIONS["2.Pattern_Analysis"]:
    cp = get_result_path("2.Pattern_Analysis/intermediate", "od_matrix.csv")
    if not FORCE_RECOMPUTE["2"] and cp.exists():
        od_matrix = pd.read_csv(cp, index_col=0, encoding="utf-8-sig")
    else:
        od_matrix = run_section_2(...)
        od_matrix.to_csv(cp, encoding="utf-8-sig")
else:
    # 章节2跳过，但章节3需要它的输出 → 直接加载
    cp = get_result_path("2.Pattern_Analysis/intermediate", "od_matrix.csv")
    od_matrix = pd.read_csv(cp, index_col=0, encoding="utf-8-sig")

# ===== Cell 2: 章节3 =====
if RUN_SECTIONS["3.Situation_Diagnosis"]:
    # od_matrix 已在上一个 cell 中加载，直接使用
    results_3 = run_section_3(od_matrix, zone_gdf, ...)
```

## 关键点

1. **中间文件放 `intermediate/` 子文件夹**，与 `star_` 论文结果分开，不混淆
2. **章节跳过时仍要加载其输出**，否则后续章节拿不到变量
3. **`index_col=0`** 读 OD 矩阵时必须加，否则行索引变成普通列，OD 对应关系会错
4. **GeoDataFrame 用 `.gpkg` 格式**，比 `.shp` 保留更多字段信息且单文件
5. **不需要封装成函数**，notebook 里直接写检查点逻辑就够，简单直接
