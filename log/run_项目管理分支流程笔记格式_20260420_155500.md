## 问题1：指标+可视化写在一起，M/V编号是否合理？

**合理**，而且对你的场景来说是更好的选择。

理由：评估的指标计算和对应可视化是强耦合的——可达性指标对应可达性地图，平衡指数对应平衡分布图。分开写反而增加维护成本。

**推荐结构**：每个 M/V 对是一个独立的 cell 块，指标计算和出图写在一起：

```python
# Cell: M1/V1 - 可达性
def eval_M1_V1_accessibility(od_matrix, zone_gdf, output_section, save_fig=True):
    """M1: 可达性指标 + V1: 可达性分布地图"""
    # 指标计算
    accessibility = od_matrix.sum(axis=1) / od_matrix.sum().sum()
    zone_gdf = zone_gdf.copy()
    zone_gdf["accessibility"] = zone_gdf["zone_id"].map(accessibility)
    
    # 保存指标
    result_csv = get_result_path(output_section, "M1_accessibility.csv")
    zone_gdf[["zone_id", "accessibility"]].to_csv(result_csv, encoding="utf-8-sig")
    
    # 可视化
    if save_fig:
        fig, ax = plt.subplots(figsize=(10, 8))
        zone_gdf.plot(column="accessibility", ax=ax, legend=True, cmap="YlOrRd")
        ax.set_title("可达性分布")
        # ... 指北针、比例尺等
        fig.savefig(get_result_path(output_section, "star_V1_accessibility_map.png"), dpi=300)
        plt.close()
    
    return zone_gdf[["zone_id", "accessibility"]]
```

**编号规则建议**：M 和 V 用同一个编号表示配对关系，如 M1/V1 是一对，M2/V2 是一对。如果一个指标有多种可视化，用 V1a、V1b 区分。

## 问题2：.ipynb 能否通过字典控制调用哪些指标和可视化？

**完全可以**，实现方式如下：

### 核心设计

在 `evaluation_dev.ipynb` 的最后一个 cell 定义注册表和主函数：

```python
# Cell: 注册表 + 主流程（放在 notebook 最后）

# 注册所有可用的评估项
EVAL_REGISTRY = {
    "M1_V1_accessibility":  eval_M1_V1_accessibility,
    "M2_V2_balance":        eval_M2_V2_balance,
    "M3_V3_distance_dist":  eval_M3_V3_distance_dist,
    "M4_V4_efficiency":     eval_M4_V4_efficiency,
}

# 预设配置
EVAL_CONFIGS = {
    "current_situation": ["M1_V1_accessibility", "M2_V2_balance", "M3_V3_distance_dist"],
    "scenario_rigid":    ["M1_V1_accessibility", "M2_V2_balance"],
    "scenario_flexible": ["M1_V1_accessibility", "M4_V4_efficiency"],
}

def run_evaluation(od_matrix, zone_gdf, output_section,
                   config_name=None, items=None, save_fig=True):
    """
    评估主函数，支持两种调用方式：
    - config_name: 使用预设配置
    - items: 直接指定要运行的编号列表
    """
    if items is None:
        items = EVAL_CONFIGS[config_name]
    
    results = {}
    for item_id in items:
        func = EVAL_REGISTRY[item_id]
        results[item_id] = func(od_matrix, zone_gdf, output_section, save_fig=save_fig)
    
    return results
```

### 在 main_pipeline.ipynb 中调用

```python
# 加载评估模块
%run notebooks/sub_pipeline/evaluation_dev.ipynb

# 方式A：用预设配置名
eval_current = run_evaluation(
    od_matrix_current, zone_gdf,
    output_section="3.Situation_Diagnosis/evaluation",
    config_name="current_situation"
)

# 方式B：直接指定编号（有微差时用这个）
eval_scenario1 = run_evaluation(
    od_matrix_s1, zone_gdf,
    output_section="4.Scenario_Computation/scenario1/evaluation",
    items=["M1_V1_accessibility", "M3_V3_distance_dist"]  # 不要 M2、M4
)

# 方式C：只算指标不出图（快速验证数据时）
eval_check = run_evaluation(
    od_matrix_s2, zone_gdf,
    output_section="4.Scenario_Computation/scenario2/evaluation",
    config_name="scenario_flexible",
    save_fig=False
)
```

### 关键点

- `%run` 执行后，notebook 中定义的所有函数和变量都进入当前 kernel 的命名空间
- 修改 `evaluation_dev.ipynb` 后，重新运行 `%run` 那个 cell 即可生效
- `items` 参数直接传编号列表，比传编号数字更直观，也不容易出错
- `save_fig=False` 可以在只需要验证数值时跳过出图，节省时间
