# 差值图右偏
create_diverging_map
你这个现象主要不是 subplots_adjust(left=...)，而是 create_diverging_map 里：

margin_x = 0.05 给左右都留了 5%
色带在 ax.inset_axes(...) 里占了右侧视觉空间
会让主图看起来“右偏”。
建议改成“左右不对称 x 边距”（左小右大），代码如下：

bounds = gdf_data_plot.total_bounds
x_span = bounds[2] - bounds[0]
y_span = bounds[3] - bounds[1]

左边少留白，右边多留白（给色带/注记）
margin_x_left = x_span * 0.01
margin_x_right = x_span * 0.06
margin_y = y_span * 0.05

ax.set_xlim(bounds[0] - margin_x_left, bounds[2] + margin_x_right)
ax.set_ylim(bounds[1] - margin_y, bounds[3] + margin_y)
ax.set_aspect('equal', adjustable='box')
并把色带略往右、略窄一点（减少对主图的“压迫感”）：
colorbar_right_edge = 0.99
cax = ax.inset_axes([0.972, 0.24, 0.018, 0.50])  # x0 + w = 0.99
然后继续用：
_add_scalebar_auto(..., right_edge_pos=colorbar_right_edge)
add_north_arrow(..., right_edge_pos=colorbar_right_edge)
这样会同时满足：

左侧空白明显变小（主图不再右偏）
比例尺/指北针/色带右端仍严格对齐。

# 前序步骤注意
add_scalebar：底层绘图函数。
负责“真正画”比例尺（线段、黑白块、0/10km文字），不关心 CRS，只按传入长度和位置画。

_add_scalebar_auto：上层包装函数。
会先判断 gdf.crs：若是地理坐标（度）就把米转换成度，再调用 add_scalebar；若是投影坐标（米）就直接调用。
也负责把 label_fontsize、right_edge_pos 这些参数传给 add_scalebar。

_prepare_map_for_plot：绘图前的数据准备步骤。
它会把 gdf_data 和 gdf_base 统一到同一个合适的投影 CRS（优先估计 UTM），避免经纬度直接画图带来的比例失真。
是 create_choropleth_map 和 create_diverging_map 的前置步骤。
