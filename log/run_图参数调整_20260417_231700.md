### 1. 修改 `create_flowline` 函数
**位置**: `src/visualization.py:521-707`

**主要改动**:
- 新增参数 `linewidth_min`, `linewidth_max` 替代固定的 `linewidth_scale`
- 新增参数 `alpha_min`, `alpha_max` 替代固定的 `alpha`
- 线宽映射：流量大 → 线宽大（`linewidth_max`），流量小 → 线宽小（`linewidth_min`）
- 透明度映射：流量大 → 不透明（`alpha_max`），流量小 → 透明（`alpha_min`）
- 逐条绘制流线以支持不同透明度（原先批量绘制无法实现逐条透明度）
- 添加线宽图例：显示最小、中间、最大流量对应的线宽，透明度不加入图例

**默认参数**:
```python

```

### 2. 修复比例尺问题
**问题**: `add_scalebar` 接收米为单位的长度（10000m），但地图坐标系为 WGS84（度），导致比例尺长度错误，图片尺寸爆炸。

**解决方案**:
- 新增 `_add_scalebar_auto(ax, gdf)` 辅助函数
- 自动判断坐标系：地理坐标系（度）时将米转换为度（纬度28°附近，1度经度≈98km）
- 投影坐标系时直接使用米
- 更新 `create_choropleth_map`, `create_diverging_map`, `create_flowline` 三处调用

## 技术要点

### 流线图线宽与透明度映射
```python
# 线宽：流量大 → 线宽大
lw = linewidth_min + (abs_flows - flow_min) / (flow_max - flow_min) * (linewidth_max - linewidth_min)

# 透明度：流量大 → 不透明
alphas = alpha_min + (abs_flows - flow_min) / (flow_max - flow_min) * (alpha_max - alpha_min)

# 逐条绘制以支持不同透明度
for i, (seg, color, width, alpha) in enumerate(zip(segments, colors, lw, alphas)):
    lc = LineCollection([seg], colors=[color], linewidths=[width], alpha=alpha)
    ax.add_collection(lc)
```

lw_bins = [flow_min, (flow_min + flow_max) / 2, flow_max]

### 比例尺坐标系自动适配
```python
def _add_scalebar_auto(ax, gdf):
    length_m = MAP_ELEMENTS['scalebar_length']  # 10000m
    if gdf.crs and gdf.crs.is_geographic:
        # 纬度28°附近，1度经度≈98km
        length_deg = length_m / 98000
        add_scalebar(ax, length_deg, x_pos, y_pos)
    else:
        add_scalebar(ax, length_m, x_pos, y_pos)
```
