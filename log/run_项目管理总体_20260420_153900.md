## 问题汇总

### 问题1: 主流程与分支流程的设计困境
**核心矛盾**: 
- main_pipeline 从原始数据运行到尾，但中间调试时已有中间结果，重跑浪费时间
- 每次局部修改都写 sub_pipeline 太繁琐，本质只是主流程切片

**解决方案**:

#### 1.1 主流程设计原则
**是的，软件工程中主流程应该可以从头运行到尾**，但需要设计**检查点机制**:

```python
# 在 main_pipeline 中的实现模式
def run_section_3_diagnosis(force_recompute=False):
    """
    3. Situation_Diagnosis 主流程
    
    Args:
        force_recompute: 是否强制重新计算（忽略缓存）
    """
    # 检查点：尝试加载中间结果
    intermediate_file = get_result_path("3.Situation_Diagnosis/intermediate", "od_matrix.csv")
    
    if not force_recompute and intermediate_file.exists():
        print(f"加载中间结果: {intermediate_file}")
        od_matrix = pd.read_csv(intermediate_file, index_col=0)
    else:
        # 从头计算
        od_matrix = compute_od_from_raw_data(...)
        od_matrix.to_csv(intermediate_file)
    
    # 继续后续步骤
    result = analyze_od_pattern(od_matrix)
    return result
```

**关键设计**:
- 每个大章节（一级标题）是一个函数，可独立调用
- 函数内部保存关键中间结果到 `results/{section}/intermediate/` 
- 默认检查中间文件是否存在，存在则跳过前置步骤
- 提供 `force_recompute` 参数强制重算

#### 1.2 开关实现方式
```python
# main_pipeline.ipynb 的结构
# Cell 1: 配置开关
RUN_SECTIONS = {
    "1.Data_Preparation": False,      # 已完成，跳过
    "2.Pattern_Analysis": False,       # 已完成，跳过
    "3.Situation_Diagnosis": True,     # 当前调试
    "4.Scenario_Computation": False,   # 未开始
}

# Cell 2: 按开关执行
if RUN_SECTIONS["3.Situation_Diagnosis"]:
    from src.diagnosis import run_section_3_diagnosis
    results_3 = run_section_3_diagnosis(force_recompute=False)
```

#### 1.3 sub_pipeline 的定位
**sub_pipeline 不是主流程切片，而是功能模块封装**:

- **main_pipeline**: 论文章节的线性流程（1→2→3→4）
- **sub_pipeline**: 跨章节复用的功能模块（如评估模块在3、4章都调用）

**示例**: 评估模块的封装
```python
# src/evaluation/pipeline.py
def run_evaluation_pipeline(
    od_matrix: pd.DataFrame,
    zone_gdf: gpd.GeoDataFrame,
    metrics: List[str] = ["accessibility", "balance", "efficiency"],
    output_section: str = "3.Situation_Diagnosis/evaluation"
) -> Dict[str, Any]:
    """
    评估子流程：可在不同章节复用
    
    Args:
        metrics: 要计算的指标列表（支持选择性调用）
        output_section: 结果输出路径
    """
    results = {}
    
    if "accessibility" in metrics:
        results["accessibility"] = compute_accessibility(od_matrix, zone_gdf)
        save_accessibility_map(results["accessibility"], output_section)
    
    if "balance" in metrics:
        results["balance"] = compute_job_housing_balance(od_matrix)
        save_balance_chart(results["balance"], output_section)
    
    # ... 其他指标
    
    return results
```

### 问题3: 个性化调用的参数设计

**推荐方案**: 使用配置字典 + 指标编号

```python
# src/evaluation/config.py
EVALUATION_CONFIGS = {
    "current_situation": {
        "metrics": ["M1_accessibility", "M2_balance", "M3_distance"],
        "visualizations": ["V1_choropleth", "V2_bar_chart"],
        "thresholds": {"accessibility": 0.5}
    },
    "scenario1_rigid": {
        "metrics": ["M1_accessibility", "M2_balance"],  # 不要 M3
        "visualizations": ["V1_choropleth", "V3_comparison"],  # 不同的图
        "thresholds": {"accessibility": 0.6}  # 不同的阈值
    },
    "scenario2_flexible": {
        "metrics": ["M1_accessibility", "M4_efficiency"],
        "visualizations": ["V1_choropleth", "V4_heatmap"],
        "thresholds": {"efficiency": 0.8}
    }
}

# src/evaluation/pipeline.py
def run_evaluation_pipeline(
    od_matrix: pd.DataFrame,
    zone_gdf: gpd.GeoDataFrame,
    config_name: str = "current_situation",  # 预设配置名
    custom_config: Dict = None,              # 或自定义配置
    output_section: str = None
) -> Dict[str, Any]:
    """
    评估子流程：通过配置名或自定义配置实现个性化调用
    """
    # 加载配置
    if custom_config:
        config = custom_config
    else:
        config = EVALUATION_CONFIGS[config_name]
    
    results = {}
    
    # 按配置计算指标
    for metric_id in config["metrics"]:
        if metric_id == "M1_accessibility":
            results["accessibility"] = compute_accessibility(od_matrix, zone_gdf)
        elif metric_id == "M2_balance":
            results["balance"] = compute_balance(od_matrix)
        # ... 其他指标
    
    # 按配置生成可视化
    for vis_id in config["visualizations"]:
        if vis_id == "V1_choropleth":
            save_choropleth_map(results, output_section)
        elif vis_id == "V2_bar_chart":
            save_bar_chart(results, output_section)
        # ... 其他图表
    
    return results
```

```python

)

    },
```

```python
# src/visualization/quick_plot.py
def quick_choropleth(gdf, column, title, output_path):
    """
    快速出图：不追求完美，只验证数据正确性
    - 不调整图例位置
    - 不精细调整字体大小
    - 不添加指北针、比例尺

    fig, ax = plt.subplots(figsize=(10, 8))
    gdf.plot(column=column, ax=ax, legend=True, cmap='YlOrRd')
    ax.set_title(title)
    plt.savefig(output_path, dpi=150, bbox_inches='tight')
    plt.close()
```

#### 4.2 出图工作流

**时间紧迫下的策略**:

1. **开发阶段**: 所有图都用 `quick_plot` 函数
   - 目的：验证数据计算正确
   - 输出到 `results/{section}/draft/` 文件夹
   - **不加 `star_` 前缀**

2. **人工检查**: 你查看草图，确认数据无误

3. **定稿阶段**: 只对论文要引用的图调用 `publication_plot`
   - 一次性批量处理所有 `star_` 图
   - 集中调整美学参数（字体、颜色、布局）
   - 输出到 `results/{section}/` 根目录

### 4. 出图 = 两阶段策略

### 5. 时间管理 = 聚焦核心
- 优先保证主流程跑通
