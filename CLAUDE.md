## 质量红线 (绝不违反)
### 数据正确性 (最高优先级)
- **必须验证**: 矩阵格式类数据中的索引是否对应其 OD 起终点
- **禁止**: 不验证就进入下一步

```python
from src import validate_od_consistency
validate_od_consistency(O, D, T)

assert T_int_df['人数'].sum() == sum(O), "整数化后总量不匹配"
```
### 代码规范
- 所有函数必须有类型注解和 docstring
- 使用 `StatsCollector` 替代 print 输出，统计信息保存到对应章节文件夹
- 错误处理: 捕获具体异常类型，记录后抛出
- 路径使用 `pathlib.Path`，禁止硬编码字符串
- 禁止使用 emoji（代码、注释、文档、commit 消息均不得使用）

### 可视化标准
- 必须包含: 指北针、比例尺、色标、标题
- 配色及字号: 使用 `config.COLOR_SCHEMES` 中定义的方案
- 输出: 300 DPI，保存为 PNG

## 紧急处理原则
1. **数据格式不匹配**: 立即停止，打印前5行检查列名、索引、数据类型
2. **模型不收敛**: 检查初始值，降低收敛阈值或增加迭代次数，保存中间结果
3. **时间不足**: 优先保证主流程跑通，注释掉待完善代码，不要删除
4. **结果异常**: 与预期对比，检查数据预处理步骤，用小样本验证逻辑

## 项目架构管理
调整 `src/` 中的函数后，必须同步更新：
1. 对应模块文件
2. `src/__init__.py`（如有新增顶层导出）
3. `notebooks/01_main_pipeline.ipynb`（调用方式）,除了特别申明的临时需求运行及sub_pipeline之外，对调用函数的流程有改动就要更新到`notebooks/01_main_pipeline.ipynb`该文件中，且按照其步骤规范地放到相应子文件夹之下。
4. 测试从主流程调用函数能否正确输出结果
